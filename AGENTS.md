이 문서는 LightMeterApp 프로젝트를 진행하는 AI 코딩 에이전트가 항상 먼저 참고해야 하는 작업 지침입니다.

2. `README.md`에 적힌 앱 목표, MVP 범위, 노출 계산 방향, Android 기술 방향을 기준으로 구현합니다.
3. 현재 코드 상태를 확인한 뒤 기존 구조와 스타일을 유지하며 변경합니다.
4. 변경 후 가능한 범위에서 빌드 또는 테스트를 실행해 검증합니다.

## 프로젝트 목표 요약

LightMeterApp은 내장 노출계가 없는 필름카메라, 수동 카메라, 빈티지 렌즈 사용자를 위한 Android 라이트 미터 앱입니다.

핵심 목표는 휴대폰 카메라 프리뷰를 이용해 장면의 노출을 측정하고, 조리개값, 셔터스피드, ISO 조작에 따른 노출 변화를 사용자가 이해하고 보정할 수 있게 돕는 것입니다.

## 우선 구현 방향

- Android 앱을 우선 개발합니다.
- 1차 배포물은 Android 직접 설치용 APK입니다.
- 앱 구조는 상단 라이브뷰 / 하단 조작부로 나눕니다.
- 조작부의 기본 상호작용은 세로 스와이프입니다.
- 원형 다이얼, AUTO 버튼, P / S / A / M 모드는 이후 확장 후보입니다.
- 현재 조작 구현에서는 고정 프리셋 값만 사용하고, 노출 기준은 장면 `EV100 16.0`을 기본 가정으로 둡니다.
- 화면 레이아웃은 프레임 기준 비율로 고정하고, 조작부는 좌/중/우 3개의 세로 카드로 배치합니다.
- 노출 계산은 표준 EV100 공식과 ISO 보정을 사용합니다.
- 조리개는 표준 f-stop 계열을 따르며, 한 스톱 변화는 f-number가 `sqrt(2)`배씩 바뀌고 빛의 양은 2배 또는 1/2배로 변하며 EV를 1단계 이동시킵니다.
- 초기 MVP는 다음 순서로 확장합니다.
  - 기본 앱 실행 및 초기 UI
  - CameraX 기반 카메라 프리뷰
  - 프레임 밝기 분석
  - EV 및 스톱 차이 계산
  - 조리개/셔터스피드/ISO 세로 스와이프 조작
  - M 모드 수동 노출 변화
  - A/S/P 모드 자동 계산

## 개발 원칙

- 노출 계산 로직은 UI 및 카메라 연동과 분리해 테스트 가능하게 유지합니다.
- 수동 필름카메라의 조작 경험을 우선합니다.
- 자동 보정 기능은 실제 카메라 모드의 이해를 돕는 가상 모드로 구현합니다.
- 사용자가 한 값을 바꾸었을 때 전체 노출이 어떻게 변하는지 명확하게 보여줍니다.
- 카메라 프리뷰는 하단 조작부의 설정값을 반영하는 방향으로 설계합니다.
- 카메라 프리뷰와 측광 결과는 사용자가 실제 촬영 설정을 판단하는 데 도움이 되도록 설계합니다.

## 라이브 카메라 작업

현재 라이브뷰 구현은 휴대폰 카메라를 직접 사용한다.

- 작업을 시작할 때는 반드시 `README.md`를 먼저 읽고, 그 내용과 맞춰 구현한다.
- 앱 실행 시 카메라 권한을 런타임으로 요청한다.
- 라이브뷰 표시는 CameraX의 `PreviewView`를 사용한다.
- Camera2 interop으로 프리뷰 메타데이터를 읽고, 가능한 범위에서 요청값을 전달한다.
- 기기가 수동 센서 제어를 지원하면 실제 조리개, 셔터스피드, ISO, 무한대 초점 제어를 우선한다.
- 기기가 수동 센서 제어를 지원하지 않으면 프리뷰는 계속 유지하고, UI에서 노출 반응을 시뮬레이션한다.
- 사용자가 별도 요청을 하지 않는 한 조작부는 고정 프리셋 스와이프 카드 구조를 유지한다.

## 현재 구현 상태

- 현재 라이브 카메라 프리뷰는 상단 패널에 연결되어 있고, 실제 기기에서 동작이 확인되었다.
- 실행 흐름에 런타임 카메라 권한 요청이 포함되어 있다.
- 셔터스피드와 ISO 스와이프 카드는 지원 기기에서 실제 카메라 동작에 반영된다.
- 테스트한 기기에서는 조리개가 고정 조리개로 보이며, 조리개 변경은 실제 노출에 반영되지 않았다.
- 테스트한 기기에서는 ISO가 대략 `3200` 정도까지는 반응하지만, 그 이상은 기기별 하드웨어 또는 드라이버 한계로 보이는 변화가 거의 없다.
- 앞으로는 현재의 프리뷰 + 조작부 레이아웃을 유지하면서, 어떤 값이 실제 하드웨어 제어인지 어떤 값이 시뮬레이션인지 더 분리해 나간다.
- 현재 앱은 물리 노출과 가상 노출을 분리해 계산한다. 휴대폰이 실제로 바꿀 수 없는 조리개 같은 항목은 가상 노출로 반영하고, 실제 카메라가 만든 밝기와의 차이만큼 프리뷰를 덮어 쓴다.
- 심도표현이나 배경 흐림은 별도 후처리 영역으로 보고, 지금 단계에서는 노출 시뮬레이션을 우선한다.
- 가상 노출이 바뀌면 실제 ISO를 먼저 보정하고, ISO 범위를 넘으면 셔터스피드를 보정한다. 그 다음에도 남는 차이만 프리뷰 밝기로 처리한다.

## 조작부 입력 규칙

- 조작부의 세로 스와이프 카드는 한 번의 제스처에 한 단계만 이동하도록 유지합니다.
- 짧은 스와이프와 긴 스와이프를 모두 같은 1단계 이동으로 처리합니다.
- 입력 감도를 조정할 때도 여러 칸이 한꺼번에 넘어가는 동작은 허용하지 않습니다.

## 프리뷰 반영 순서

- 프리뷰는 먼저 실제 물리 노출이 바뀐 결과를 보여줍니다.
- 가상 노출 보정은 카메라 메타데이터가 갱신된 뒤에만 적용합니다.
- 조작 직후의 화면 반영은 물리값 우선, 가상값 후순위 원칙을 따릅니다.
