# LightMeterApp

> 이 문서와 `AGENTS.md`를 포함한 프로젝트의 모든 `.md` 문서는 한국어로 작성한다. 새로 추가하거나 갱신하는 설명, 상태 정리, 작업 메모도 모두 한국어를 기본으로 한다.

내장 노출계가 없는 필름카메라, 수동 카메라, 빈티지 렌즈 사용자를 위한 Android용 라이트 미터 앱입니다.

휴대폰 카메라 화면을 이용해 현재 장면의 노출을 측정하고, 조리개값, 셔터스피드, ISO를 직접 조작하면서 실제 수동 카메라에서 발생하는 노출 변화를 이해하고 보정할 수 있도록 돕는 것을 목표로 합니다.

> Android 직접 설치 배포 형식은 일반적으로 `.apk`입니다. iOS/macOS 계열의 `.app`과 구분되므로, 본 프로젝트의 1차 배포 목표는 Android용 APK 파일로 정의합니다.

## 프로젝트 목표

LightMeterApp의 궁극적인 목표는 필름카메라처럼 내장 노출계가 없거나, 수동 노출 조작이 필요한 카메라의 촬영 설정을 보조하는 것입니다.

디지털 카메라는 조리개, 셔터스피드, ISO 중 하나를 바꾸면 다른 값이 자동으로 보정되어 노출을 유지하는 경우가 많습니다. 반면 수동 필름카메라는 한 값을 바꾸면 전체 노출이 그대로 변하며, 사용자가 직접 다른 값을 보정해야 합니다.

이 앱은 두 가지 흐름을 모두 제공합니다.

- 등가노출 원리에 따라 노출값을 유지하며 다른 요소가 자동 보정되는 흐름
- 수동 필름카메라처럼 한 요소 변경 시 전체 노출이 변하고 사용자가 직접 보정하는 흐름

또한 휴대폰 카메라의 실시간 프리뷰를 통해 사용자가 실제 촬영 결과에 가까운 화면을 보고 판단할 수 있도록 합니다.

## UI 구조

앱은 상단의 라이브뷰 영역과 하단의 조작부 영역으로 나눕니다.

### 라이브뷰 영역

- 앱 상단에 배치합니다.
- 휴대폰 후면 카메라를 기본 프리뷰 소스로 사용합니다.
- 사용자가 원하면 다른 카메라 소스를 선택할 수 있도록 확장합니다.
- 하단 조작부에서 정한 조리개, 셔터스피드, ISO 값이 반영된 프리뷰를 보여주는 방향으로 설계합니다.
- 초기에는 리소스 절감을 위해 FPS 제한이나 분석 주기 제한을 고려할 수 있습니다.

### 조작부 영역

- 앱 하단에 배치합니다.
- 조리개, 셔터스피드, ISO 등의 노출 값을 조절합니다.
- 기본 조작 방식은 세로 스와이프입니다.
- 추후 원형 다이얼 UX를 추가할 수 있습니다.

이 프로젝트의 1차 우선순위는 세로 스와이프 기반 조작을 안정적으로 구현하는 것입니다.

## 핵심 기능

### 1. 카메라 기반 측광

- 휴대폰 카메라 프리뷰를 통해 구도를 잡습니다.
- 현재 화면 또는 지정된 측광 영역의 밝기를 분석합니다.
- 분석 결과를 EV(Exposure Value) 또는 보정된 노출 지표로 숫자 출력합니다.


### 2. 노출 3요소 조작

사용자는 다음 값을 직접 조작할 수 있습니다.

- 조리개값: 예 `f/1.4`, `f/2`, `f/2.8`, `f/4`, `f/5.6`, `f/8`, `f/11`, `f/16`
- 셔터스피드: 예 `1/1000`, `1/500`, `1/250`, `1/125`, `1/60`, `1/30`, `1/15`, `1s`
- ISO: 예 `ISO 50`, `100`, `200`, `400`, `800`, `1600`, `3200`

각 값은 1스톱 단위부터 시작하고, 이후 1/2스톱 또는 1/3스톱 단위 지원을 고려합니다.
조리개는 표준 f-stop 계열을 따르며, 한 스톱 변화는 f-number가 `sqrt(2)`배씩 바뀌고 빛의 양은 2배 또는 1/2배로 변하며 EV는 1단계 이동합니다.

### 3. 등가노출 및 수동 노출 변화

앱은 두 가지 계산 방식을 지원합니다.

#### 등가노출 유지

하나의 요소를 변경하면 나머지 요소 중 하나가 자동 보정되어 전체 노출값을 유지합니다.

예:

- `f/4, 1/125, ISO 100`
- 조리개를 `f/5.6`으로 1스톱 어둡게 변경
- 셔터스피드를 `1/60`으로 1스톱 밝게 보정
- 전체 노출값 유지

#### 수동 노출 변화

하나의 요소를 변경하면 전체 노출값이 변합니다. 사용자는 직접 다른 요소를 조정해 목표 노출로 돌아가야 합니다.

예:

- `f/4, 1/125, ISO 100`
- 조리개를 `f/5.6`으로 변경
- 전체 노출은 1스톱 어두워짐
- 사용자가 셔터스피드 또는 ISO를 직접 보정

이 흐름은 실제 수동 필름카메라 조작 경험에 가깝게 설계합니다.

### 4. 가상 카메라 모드

휴대폰 카메라의 실제 촬영 모드가 아니라, 앱 내부에서 노출 계산 방식을 제어하는 가상 카메라 모드입니다.

| 모드 | 의미 | 사용자가 주로 조작하는 값 | 앱의 동작 |
| --- | --- | --- | --- |
| P | Program | 노출 보정, ISO 등 | 앱이 조리개/셔터 조합을 제안 |
| A | Aperture Priority | 조리개 | 셔터스피드를 자동 계산 |
| S | Shutter Priority | 셔터스피드 | 조리개값을 자동 계산 |
| M | Manual | 조리개, 셔터스피드, ISO | 자동 보정 없이 노출 변화 표시 |

M 모드는 필름카메라 사용자에게 가장 중요한 모드로, 값 변경에 따른 노출 초과/부족을 명확하게 보여주는 것을 우선합니다.

초기 MVP에서는 우선 세로 스와이프 조작만 구현하고, 모드 전환이나 자동 보정은 이후 단계에서 붙입니다.

## 노출 계산 개념

기본 노출 계산은 EV를 중심으로 설계합니다.

```text
EV100 = log2(N^2 / t)
EV = EV100 - log2(ISO / 100)
```

- `N`: 조리개값
- `t`: 셔터 시간(초)
- `ISO`: 감도

카메라 프리뷰에서 측정한 장면의 기준 EV와 사용자가 설정한 조리개/셔터/ISO 조합의 EV를 비교해 다음 정보를 표시합니다.

- 현재 설정이 적정 노출인지
- 몇 스톱 과다 노출인지
- 몇 스톱 부족 노출인지
- 어떤 값을 얼마나 조정하면 목표 노출에 가까워지는지

## 프리뷰 방향

휴대폰 카메라의 자동 노출 결과를 그대로 신뢰하지 않고, 사용자가 설정한 가상 카메라 값에 따른 결과를 화면에서 판단할 수 있도록 하는 것이 중요합니다.

초기 버전에서는 다음 방식으로 접근합니다.

1. CameraX로 상단 라이브뷰 영역에 실시간 프리뷰를 표시합니다.
2. 이미지 분석 프레임에서 평균/가중 밝기를 계산합니다.
3. 측정 EV와 사용자 설정 EV의 차이를 계산합니다.
4. 프리뷰는 하단 조작부의 현재 설정값을 반영하도록 설계합니다.
5. 프리뷰 위에 노출 차이를 숫자와 간단한 시각 표시로 제공합니다.

향후에는 실제 프리뷰 밝기를 시뮬레이션하는 톤 매핑 또는 오버레이를 추가합니다.

## 카메라 구현 상태

현재 앱은 휴대폰 카메라를 상단 라이브뷰 영역의 실시간 프리뷰로 사용한다.

- 앱 시작 시 카메라 권한을 런타임으로 요청한다.
- 프리뷰는 CameraX의 `PreviewView`로 구성한다.
- Camera2 interop을 사용해 캡처 메타데이터를 읽고, 가능한 범위에서 카메라 요청값을 반영한다.
- 기기가 수동 센서 제어를 지원하면 조리개, 셔터스피드, ISO, 무한대 초점 요청을 카메라 스택에 전달한다.
- 기기가 해당 제어를 지원하지 않으면 프리뷰는 계속 유지하고, 선택한 설정이 눈에 보이도록 간단한 노출 시뮬레이션을 적용한다.
- 조작부는 여전히 고정 프리셋 기반이며, 현재 EV100 흐름은 장면 밝기와 스톱 차이 비교를 중심으로 유지한다.

## 현재 구현 메모

현재 Android 빌드는 실제 기기에서 테스트되었고, 라이브 카메라 흐름이 동작하는 상태다.

- 카메라 권한은 실행 시 요청된다.
- 상단 라이브뷰 패널에는 후면 카메라 프리뷰가 표시된다.
- 하단 조작부는 조리개, 셔터스피드, ISO에 대해 세로 스와이프 카드를 계속 사용한다.
- 지원되는 기기에서는 셔터스피드와 ISO 변화가 실제 카메라 동작에 반영된다.
- 테스트한 기기에서는 조리개가 고정 조리개로 보이며, 조리개 카드를 바꿔도 실제 노출은 변하지 않았다.
- 테스트한 기기에서는 ISO가 대략 `3200` 정도까지는 반응했지만, 그 이상은 하드웨어 또는 드라이버 한계로 보이는 변화가 거의 없었다. 상위 프리셋 값은 UI에 보여줄 수 있지만 실제 노출 변화로 이어지지 않을 수 있다.
- 현재 앱은 물리 노출과 가상 노출을 분리해서 계산한다. 조리개처럼 휴대폰이 실제로 바꾸지 못하는 항목은 가상 노출로 반영하고, 물리 카메라가 실제로 만든 값과의 차이만큼 프리뷰 밝기를 덮어 쓴다.
- 실제 카메라 앱의 심도표현이나 배경 흐림은 보통 물리 조리개가 아니라 깊이 정보, 인물 분리, 후처리 블러로 구현되므로, 이 앱의 가상 조리개는 우선 프리뷰 노출 시뮬레이션에 집중한다.
- 현재는 가상 노출이 바뀌면 실제 카메라 쪽에서 먼저 요청한 셔터스피드를 유지하려 시도하고, ISO가 지원 범위를 벗어나면 셔터스피드도 같은 방향에서 한 단계 이상 이동하도록 강제한다. 이때 카메라 메타데이터 범위는 참고만 하고, 사용자 요청값을 우선으로 삼는다.
- 그 다음에도 부족할 때만 프리뷰 밝기를 마지막 수단으로 조정한다.

## 다음 작업 방향

- 현재의 라이브뷰 + 세로 스와이프 조작 구조를 유지한다.
- 조리개는 고정 조리개 휴대폰에서 시뮬레이션 전용 경로로 전환하는 방향을 검토한다.
- ISO 프리셋은 기기 지원 범위를 반영해 제한하거나, 범위를 넘는 값은 명확히 표시한다.
- 카메라 메타데이터는 계속 읽어 두어, 더 나은 노출 보조 도구로 발전할 수 있게 한다.
- 심도표현이나 배경 흐림을 넣을 때는 별도 후처리 효과로 다루고, 먼저는 노출 시뮬레이션의 정확도를 더 높인다.
- 가상 조리개 변화는 먼저 실제 셔터 반영과 ISO 보정을 통해 흡수하고, 지원 범위를 벗어나는 부분만 프리뷰 보정으로 처리한다.

## 조작부 입력 규칙

- 조리개, 셔터스피드, ISO 조작 카드는 한 번의 세로 스와이프에 정확히 한 단계만 반응합니다.
- 짧게 밀어도 길게 밀어도 이동량은 항상 1칸으로 고정합니다.
- 스와이프 방향만 보고 증감 방향을 판단하고, 거리 차이로 여러 단계가 한꺼번에 넘어가지 않도록 유지합니다.

## 프리뷰 반영 순서

- 카메라 프리뷰는 우선 실제 물리 노출 변화를 반영합니다.
- 가상 노출 보정은 카메라가 실제로 갱신한 물리 상태가 들어온 뒤에만 덧씌웁니다.
- 조작 직후에는 화면이 먼저 물리값을 따라가고, 그 이후에 필요한 경우에만 가상 보정이 보이도록 유지합니다.
- 수동 제어가 가능한 기기에서는 요청한 셔터스피드를 우선 실제 카메라에 반영하고, ISO가 지원 범위를 벗어나면 요청 방향과 같은 쪽으로 셔터를 한 단계 이상 반드시 다시 조정합니다.

## 조작 단계 선택

- 조작부 상단에는 단계 선택 드롭다운이 있습니다.
- 현재 선택 가능한 항목은 `1stop`, `1/2stop`, `1/3stop`입니다.
- 각 항목은 조리개, 셔터스피드, ISO 카드에 표시되는 눈금 범위를 바꿉니다.
- 1stop은 기존 1스톱 눈금을, 1/2stop과 1/3stop은 더 촘촘한 눈금을 사용합니다.
- ISO는 단계가 더 촘촘할수록 상단에서 더 넓은 값 범위를 보여줍니다.

## 1.0.0 릴리즈 메모

- 릴리즈 버전에서는 프리뷰 영역의 노출값, 가상 노출값, 상태 문구를 화면에서 제거했습니다.
- 배포용 화면은 실제 카메라 프리뷰와 조작부에 집중하도록 정리했습니다.
- 개발 중 세부 노출 표시는 debug 빌드에서만 확인하도록 유지합니다.
